# 01 — 요구사항 (요약)

## 기능

1. **대역폭 로깅**: `AXI SIZE`·`LEN`으로 바이트 계산; period 종료 시 write/read 바이트를 FIFO에 push. Period는 **사이클 수** 또는 **필터 통과 AW+AR 핸드셰이크 수** 중 선택.
2. **Even/Odd 뱅크**: FIFO depth는 파라미터; **N** period마다 쓰기 뱅크 전환(`N < FIFO_DEPTH` 권장). 전환 시 인터럽트(옵션).
3. **지연 로깅**: 주기마다 평균만 저장—write는 ID별 큐에서 AW→B, read는 AR→첫 R / AR→마지막 R.
4. **주소 필터**: `ADDR_FILTER`가 켜지면 `[START, START+SIZE)` 안의 트랜잭션만 추적; `SIZE==0`이면 필터 ON 시 매칭 없음.
5. **스로틀**: 별도 `THR_PERIOD`마다 동일 바이트 계산으로 합산; 상한 초과 시 **필터 영역**에 한해 AW/AR 수락 후 갭 삽입.

## 비기능

- 상·하류 **동일 데이터 폭**, 기본 **256bit**.
- APB **32bit**, 인터럽트 **1개**.
- 시뮬레이션은 사용자 환경에서 수행(본 저장소에서는 시뮬 실행 가정 없음).

## 추적 정책 (현재 RTL)

- 바이트/카운트: **AW/AR 주소**가 필터에 들어올 때만 카운트.
- 스로틀: **필터와 동일한 주소 범위**로만 바이트 합산 및 갭 적용.
