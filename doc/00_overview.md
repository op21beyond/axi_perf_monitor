# 00 — 개요

## 목적

이 IP는 AXI4 트래픽을 **투명하게 통과**시키면서, 동일 클럭의 **APB 32비트**로 제어·상태를 읽고 다음을 수행한다.

- **대역폭 로깅**: 주기(사이클 또는 트랜잭션 수)마다 read/write 바이트 합을 Even/Odd FIFO에 저장
- **지연 로깅**: 주기마다 AW→B, AR→첫 R, AR→마지막 R의 **평균 지연(사이클)** 을 Even/Odd FIFO에 저장
- **대역폭 스로틀**: 별도 주기 내 바이트 합이 상한을 넘으면, **주소 필터로 지정한 영역**에 대해서만 `awready`/`arready`에 갭을 삽입

## 인터페이스

| 인터페이스 | 역할 |
|------------|------|
| `up_*` (top) | 상류 마스터가 붙는 **슬레이브** 포트 |
| `dn_*` (top) | 하류 메모리/슬레이브로 향하는 **마스터** 포트 |
| APB | CSR, `pstrb=4'b1111` 가정, AXI와 동일 클럭 |
| `irq` | 레벨 인터럽트 1개(인터럽트 enable 및 클리어는 CSR) |

## RTL 계층

- `axi_perf_monitor_top`: optional `USE_SLICE_UP` / `USE_SLICE_DN` (현재 slice는 조합 패스스루; 벤더 slice로 교체 가능)
- `axi_perf_monitor_core`: CSR + 모니터 + 스로틀

## 문서 읽는 순서

`01` 요구사항 → `02` 마이크로아키텍처 → `03` 파라미터/제약 → `04` 검증 계획 → `05` RTL 가이드 → `06` 트레이드오프 → `07` SoC 통합 → `08` 레지스터 맵 → `09` 소프트웨어 가이드.
